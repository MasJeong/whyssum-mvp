# 왜씀? 웹사이트형 요건 정의 (현재 반영)

## 1) 제품/콘텐츠 요건

- 직무별 진입이 즉시 가능해야 함 (`/roles`, 홈 직무 카드)
- 트렌드-상황추천-비교의 의사결정 흐름이 연결돼야 함
- 근거 기반 서비스임을 명시해야 함 (출처/표본/업데이트 안내)
- 신규 사용자도 이해 가능한 FAQ/설명 섹션 필요

## 2) UX 요건

- 홈은 랜딩 페이지 구조(히어로, 가치 제안, 진입 CTA, 신뢰 요소, FAQ)
- 모바일에서 깨지지 않는 내비게이션 필수
- 하단 탭/스티키 CTA는 화면 가림 없이 동작해야 함
- 현재 페이지(active) 식별이 명확해야 함

## 3) 기능 요건

- 상황추천은 정적 카드가 아니라 조건 기반 재계산 가능해야 함
- 비교는 고정 데이터가 아니라 직무별 전환 + 선택형 비교가 가능해야 함
- 추천/비교 흐름에서 에러/빈값 안내가 있어야 함
- 트렌드 데이터는 실데이터 소스 연동 + 실패 시 fallback 전략이 필요함
- 트렌드 지표는 채택률/성장률 외 활동성/커뮤니티/안정성까지 제공해야 함

## 4) 운영/신뢰 요건

- 보안 기본선 유지(CSP, rate limit, 입력 검증)
- 문서(`review_docs`)는 변경 시점마다 최신 상태 유지
- lint/build를 통과해야 반영 완료로 간주

## 이번 라운드 구현 반영

- 홈 랜딩 구조를 웹사이트형으로 재구성
- footer 문구를 단순 실험 텍스트에서 실서비스형 링크 구성으로 변경
- 모바일 깨짐 완화(상단 nav 숨김, 하단 탭 + 스티키 CTA 정리)
- 상황추천 인터랙티브 재계산 흐름 추가
- 비교 화면 멀티 직무 전환 + 선택형 비교 강화
- GitHub 공개 메타데이터 + 커밋 활동 기반 트렌드 지표 연동
