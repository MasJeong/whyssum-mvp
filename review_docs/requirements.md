# 왜씀? 웹사이트형 요건 정의 (현재 반영)

## 1) 제품/콘텐츠 요건

- 직무별 진입이 즉시 가능해야 함 (`/roles`, 홈 직무 카드)
- 트렌드-상황추천-비교의 의사결정 흐름이 연결돼야 함
- 근거 기반 서비스임을 명시해야 함 (출처/표본/업데이트 안내)
- 신규 사용자도 이해 가능한 FAQ/설명 섹션 필요

## 2) UX 요건

- 홈은 랜딩 페이지 구조(히어로, 가치 제안, 진입 CTA, 신뢰 요소, FAQ)
- 모바일에서 깨지지 않는 내비게이션 필수
- 하단 탭/스티키 CTA는 화면 가림 없이 동작해야 함
- 현재 페이지(active) 식별이 명확해야 함
- 밀집 정보 화면(트렌드/비교)은 요약 우선 + 상세 확장 구조를 유지해야 함
- 인터랙션 컴포넌트는 키보드 포커스 가시성과 최소 터치 타깃(44px+)을 보장해야 함

## 3) 기능 요건

- 상황추천은 정적 카드가 아니라 조건 기반 재계산 가능해야 함
- 상황추천은 사용자가 자주 쓰는 조건을 저장하고 재방문 시 빠르게 복원할 수 있어야 함
- 상황추천은 "자세히 보기"에서 조건 정합성 해설과 실행 체크리스트를 제공해야 함
- 상황추천은 "자세히 보기"에서 상황별 점수 변화/신뢰도/리스크를 이해하기 쉽게 단계적으로 제공해야 함
- 상세 패널은 사용자가 피로하지 않도록 요약(기본) → 확장(세부) 구조를 유지해야 함
- 비교는 고정 데이터가 아니라 직무별 전환 + 선택형 비교가 가능해야 함
- 추천/비교 흐름에서 에러/빈값 안내가 있어야 함
- 트렌드 데이터는 실데이터 소스 연동 + 실패 시 fallback 전략이 필요함
- 트렌드 지표는 채택률/성장률 외 활동성/커뮤니티/안정성까지 제공해야 함
- 신뢰도 표시(신뢰도 점수/등급)와 추천 근거 상세가 제공되어야 함
- 상황추천은 "왜 지금 이 선택인지"를 설명하는 Why now 문장과 트레이드오프(속도/안정성/확장성)를 제공해야 함
- 트렌드 화면은 Top N/시계열 탐색을 지원해야 함
- 관심리스트 저장/조회 기능이 있어야 함
- 지표 해석을 위한 브리핑(뉴스/동향) 메뉴가 있어야 함

## 4) 운영/신뢰 요건

- 보안 기본선 유지(CSP, rate limit, 입력 검증)
- 문서(`review_docs`)는 변경 시점마다 최신 상태 유지
- lint/build를 통과해야 반영 완료로 간주

## 이번 라운드 구현 반영

- 홈 랜딩 구조를 웹사이트형으로 재구성
- footer 문구를 단순 실험 텍스트에서 실서비스형 링크 구성으로 변경
- 모바일 깨짐 완화(상단 nav 숨김, 하단 탭 + 스티키 CTA 정리)
- 상황추천 인터랙티브 재계산 흐름 추가
- 비교 화면 멀티 직무 전환 + 선택형 비교 강화
- GitHub 공개 메타데이터 + 커밋 활동 기반 트렌드 지표 연동

## 이번 라운드 추가 반영 (Phase 3)

- 추천 근거 상세(`reasons`) 및 시나리오 프리셋 적용
- 트렌드 신뢰도 등급/점수 및 소스 정보 표시
- Top N 토글(5/8/12) + 6개월 시계열 시각화(스파크라인)
- 관심리스트(로컬 저장) 추가 및 전용 페이지 구현
- 데이터 소스 다변화: GitHub + npm + PyPI + catalog signal 결합

## 이번 라운드 추가 반영 (브리핑)

- 브리핑 기능 계획 문서 작성
- 브리핑 API 및 필터(role/impact/periodDays) 구현
- `/briefings` 페이지 및 카드 동선(원문/트렌드/비교/상황추천) 구현
- 상단/하단/푸터 내비게이션에 브리핑 메뉴 연결

## 이번 라운드 추가 반영 (상황추천 explainability)

- 추천 API 응답에 `confidenceScore`/`trustLevel`/`whyNow`/`tradeoff` 필드 추가
- 추천 카드에 신뢰도 배지 + Why now + 의사결정 트레이드오프(프로그레스바) 표시
- API 필드가 없을 때도 fallback 추정값으로 UX가 끊기지 않도록 처리

## 이번 라운드 추가 반영 (상황추천 재방문)

- 조건 기반 추천 카드 상단에 "현재 조건 저장" 동작 추가
- 직무별 저장 조건 목록(불러오기/삭제/전체삭제) 제공
- 직무별 마지막 선택값(teamSize/timeline/priority) 자동 복원으로 재방문 진입 시간 단축

## 이번 라운드 추가 반영 (상황추천 상세 해설)

- 추천 카드별 "자세히 보기/간단히 보기" 토글 추가
- 상세 뷰에서 조건-추천 정합성 해설, 주력 포인트, 실행 체크리스트 제공
- 상세 뷰에서 이번 계산에 반영된 조건(appliedRules) 칩 목록 제공

## 이번 라운드 추가 반영 (상황맞춤 상세 UX 정리)

- 상세 패널에서 사용자 피로를 유발하는 과도한 보조 정보 노출 제거
- 사용자 선택 조건과 연결된 상황 해설/실행 체크리스트 중심으로 상세 보기 재구성
- 기본 카드는 핵심 요약(적합도/신뢰도/판단) 위주, 근거/장단점/리스크는 확장 시 노출

## 이번 라운드 추가 반영 (UI/UX 현대화 1차)

- 의미 기반 텍스트 토큰(`text-primary/secondary`)과 타이포 위계 강화
- 전역 포커스 가시성(`:focus-visible`) 및 버튼/필 탭 최소 터치 타깃 상향
- 비교/트렌드 표 가독성 개선(첫 열 고정, 행 hover 피드백, 캡션 추가)
- 인라인 스타일 일부를 공통 유틸 클래스(`mt-*`, `stack-*`, `readable`)로 정리
