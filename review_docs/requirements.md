# 왜씀? 웹사이트형 요건 정의 (현재 반영)

## 1) 제품/콘텐츠 요건

- 직무별 진입이 즉시 가능해야 함 (`/roles`, 홈 직무 카드)
- 트렌드-상황추천-비교의 의사결정 흐름이 연결돼야 함
- 근거 기반 서비스임을 명시해야 함 (출처/표본/업데이트 안내)
- 신규 사용자도 이해 가능한 FAQ/설명 섹션 필요

## 2) UX 요건

- 홈은 랜딩 페이지 구조(히어로, 가치 제안, 진입 CTA, 신뢰 요소, FAQ)
- 모바일에서 깨지지 않는 내비게이션 필수
- 하단 탭/스티키 CTA는 화면 가림 없이 동작해야 함
- 현재 페이지(active) 식별이 명확해야 함
- 밀집 정보 화면(트렌드/비교)은 요약 우선 + 상세 확장 구조를 유지해야 함
- 인터랙션 컴포넌트는 키보드 포커스 가시성과 최소 터치 타깃(44px+)을 보장해야 함
- 핵심 의사결정 화면은 "한 줄 결론"을 상단에 배치해 사용자가 3초 내 판단할 수 있어야 함
- 모바일 고정 CTA는 현재 문맥(직무/흐름)에 맞는 경로로 연결되어야 함
- 자주 사용하는 조건 칩/프리셋은 읽기 전용이 아니라 즉시 실행 가능한 인터랙션이어야 함

## 3) 기능 요건

- 상황추천은 정적 카드가 아니라 조건 기반 재계산 가능해야 함
- 상황추천은 사용자가 자주 쓰는 조건을 저장하고 재방문 시 빠르게 복원할 수 있어야 함
- 상황추천은 "자세히 보기"에서 조건 정합성 해설과 실행 체크리스트를 제공해야 함
- 상황추천은 "자세히 보기"에서 상황별 점수 변화/신뢰도/리스크를 이해하기 쉽게 단계적으로 제공해야 함
- 상세 패널은 사용자가 피로하지 않도록 요약(기본) → 확장(세부) 구조를 유지해야 함
- 비교는 고정 데이터가 아니라 직무별 전환 + 선택형 비교가 가능해야 함
- 비교는 기본 모드(최대 4개)와 고급 모드(최대 8개)를 지원해야 함
- 추천/비교 흐름에서 에러/빈값 안내가 있어야 함
- 트렌드 데이터는 실데이터 소스 연동 + 실패 시 fallback 전략이 필요함
- 트렌드 지표는 채택률/성장률 외 활동성/커뮤니티/안정성까지 제공해야 함
- 신뢰도 표시(신뢰도 점수/등급)와 추천 근거 상세가 제공되어야 함
- 상황추천은 "왜 지금 이 선택인지"를 설명하는 Why now 문장과 트레이드오프(속도/안정성/확장성)를 제공해야 함
- 트렌드 화면은 Top N/시계열 탐색을 지원해야 함
- 관심리스트 저장/조회 기능이 있어야 함
- 지표 해석을 위한 브리핑(뉴스/동향) 메뉴가 있어야 함
- 브리핑은 재방문 시 마지막 필터(직무/영향도/기간)를 복원해 즉시 탐색 가능해야 함
- 브리핑은 상단에 핵심 요약(High 영향도/최근 7일/추천 직무)을 제공해야 함
- 트렌드 데이터는 주기 갱신을 위한 스케줄 실행 API를 제공해야 함
- 브리핑 이벤트를 주간 KPI 리포트로 요약할 수 있는 템플릿이 있어야 함

## 4) 운영/신뢰 요건

- 보안 기본선 유지(CSP, rate limit, 입력 검증)
- 문서(`review_docs`)는 변경 시점마다 최신 상태 유지
- lint/build를 통과해야 반영 완료로 간주
- TypeScript/TSX의 모든 named function/method는 한국어 중심 JSDoc을 필수로 가진다

## 이번 라운드 추가 반영 (색감/아이콘 2차)

- 주요 액션 버튼은 다색 포인트를 남발하지 않고 신뢰 블루 중심 계층(Primary/Secondary)으로 유지
- 내비게이션/핵심 CTA는 아이콘 단독이 아닌 아이콘+라벨 조합을 기본으로 사용
- 아이콘은 장식이 아닌 의미 전달(탐색/추천/비교/저장/외부링크) 역할에 한정
- 보조 액션(삭제/세부 토글)은 compact 버튼 규격으로 밀도를 낮추되 터치 영역 기준은 유지
