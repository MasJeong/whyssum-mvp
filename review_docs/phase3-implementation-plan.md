# Phase 3 구현 계획서 (신뢰도/추천근거/Top N/시계열/프리셋/관심리스트/데이터소스)

## 목표

`왜씀`을 "데이터 조회 MVP"에서 "의사결정 근거 플랫폼"으로 고도화한다.

핵심 목표:

1. 추천 결과의 신뢰/설명 가능성 강화
2. 트렌드 지표 깊이와 탐색성 강화
3. 재방문 유도 기능(관심리스트) 추가
4. 외부 데이터 소스 다변화

---

## 범위 (이번 구현)

### A. 신뢰도 표시

- 각 도구 행에 `신뢰도 등급(High/Medium/Low)` 표시
- 등급 계산 근거: 소스 가용성(깃허브/레지스트리/보조신호), 값 최신성

### B. 추천 근거 상세

- `/api/recommendations`에서 추천안별 상세 근거(`reasons[]`) 반환
- UI에서 카드마다 "왜 이 추천인지"를 텍스트로 표시

### C. 직무별 Top N 확장

- 트렌드 화면에 Top N 선택(예: 5/8/12)
- 비교 화면에도 role 전환 시 확장된 후보 반영

### D. 시계열 보기

- 도구별 최근 추세(월 단위 다중 포인트) 노출
- 우선 서버 계산 기반 시계열로 시작 (실데이터 히스토리 저장 전 단계)

### E. 맞춤 시나리오 프리셋

- 상황추천 화면에 프리셋 버튼 제공
  - 예: 1인 MVP, 팀 확장, 운영 안정화
- 프리셋 클릭 시 팀/일정/우선순위가 자동 입력되고 재계산

### F. 관심리스트

- localStorage 기반 관심 도구 저장/해제
- 트렌드/비교에서 추가 가능
- 별도 `관심리스트` 화면에서 모아보기

### G. 데이터소스 다변화

- 기존 GitHub 외 보조 소스 추가
  - npm 다운로드(가능한 도구)
  - PyPI 다운로드(가능한 도구)
  - 보조 신호(catalog signal) fallback
- 소스별 실패는 부분 fallback 처리

## 아키텍처 변경점

1. `TrendMetric` 타입 확장
   - trust/confidence, source metadata, 시계열 배열
2. `live-role-trends.ts` 고도화
   - 멀티 소스 수집 + 가중 합산
   - 부분 실패 시 per-source fallback
3. `api/trends/[role]` 응답 확장
   - metrics + mode + source + fetchedAt 유지
4. `api/recommendations` 응답 확장
   - 카드별 rationale/reasons 포함
5. 클라이언트 상태 추가
   - topN 선택 상태
   - watchlist 상태(localStorage)

## 구현 순서

1. 추천 API/시나리오 UI 안정화(근거 상세 + 프리셋)
2. 트렌드 데이터 모델 확장(신뢰도/시계열/멀티소스)
3. 트렌드/비교 UI 반영(Top N, 지표 컬럼, 신뢰도)
4. 관심리스트 기능 및 화면 추가
5. 문서 동기화 + lint/build + 커밋/푸시

---

## 수용 기준 (Definition of Done)

- 추천 화면에서 조건 변경 시 결과 점수/순위가 실제로 변화한다.
- 추천 카드에 근거 상세가 표시된다.
- 트렌드 화면에서 Top N 변경이 반영된다.
- 트렌드 화면에 시계열 정보가 표시된다.
- 트렌드/비교 화면에서 신뢰도 표시가 노출된다.
- 관심리스트 추가/삭제/목록 확인이 가능하다.
- 멀티 소스 실패 시 전체 장애 없이 fallback으로 동작한다.
- `npm run lint`, `npm run build` 통과.
- `README.md`, `review_docs/*` 최신화.

## 리스크 및 대응

- 외부 API rate limit: 캐시 + 부분 fallback + source별 가중치 하향
- 이질 소스 정규화 난이도: 공통 0~100 스케일로 정규화
- 과도한 UI 복잡성: 기본 뷰는 간결, 상세는 확장 섹션으로 분리
- 1인 운영 부담: 로컬 관심리스트부터 시작하고 서버 동기화는 후속

---

## 후속 과제 (이번 범위 밖)

- 관심리스트 계정 연동(로그인 기반)
- 실제 히스토리 저장(DB)
- 알림(주간 리포트/관심 항목 변화)
- 분산 rate limit(Redis) 및 관측성 강화
