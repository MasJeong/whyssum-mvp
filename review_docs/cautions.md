# 주의사항 및 확인 포인트

## 기능 동작

- 모바일 하단 탭/스티키 CTA는 `max-width: 720px` 구간에서만 노출됩니다.
- 비교 선택 UI는 최소 1개, 최대 4개로 제한되어 있습니다.
- 트렌드/상황추천의 role active 상태는 경로 prefix 매칭 기준입니다.
- 상황추천은 조건 변경 시 자동 재계산 + 수동 재계산 버튼이 함께 동작합니다.
- 추천 결과에는 카드별 "추천 근거"가 함께 표시됩니다.
- 트렌드 페이지 Top N는 5/8/12 선택 가능하나, 실제 표시는 해당 role 데이터 개수 상한을 따릅니다.
- 관심리스트는 브라우저 localStorage 기반이며 기기/브라우저 간 동기화되지 않습니다.

## 디자인/접근성

- 컬러 대비를 다크 배경/저밝기 기기에서 추가 검토 권장
- 모바일에서 하단 탭 + 스티키 CTA 동시 노출로 콘텐츠 가림이 없는지 확인 필요
- 체크박스 카드의 키보드 탐색 포커스 스타일(명확성) 추가 개선 여지 있음
- FAQ/랜딩 섹션 텍스트 길이가 늘어나 모바일에서 스크롤 피로가 없는지 확인 필요

## 보안/운영

- 기존 보안 미들웨어와 API rate limit은 유지됨
- 현재 rate limit은 메모리 기반이라 다중 인스턴스/재시작 환경에서 약함
- 운영 전 Redis 기반 분산 rate limit 전환 권장
- GitHub 공개 API는 무인증 호출 시 rate limit이 낮아 트래픽 증가 시 fallback 빈도가 올라갈 수 있음
- 일부 repo가 일시 실패해도 전체 fallback으로 내려가지 않도록 부분 fallback이 적용되어 있습니다.
- npm/PyPI 보조 소스 호출 실패 시 해당 소스만 제외하고 계산합니다.

## QA 권장 시나리오

1. 모바일 뷰(360px~430px)에서 하단 네비/CTA 동작 확인
2. 비교 페이지에서 4개 초과 선택 차단 확인
3. `/trends/[role]`, `/scenarios/[role]` role 전환 active 상태 확인
4. 전체 페이지에서 버튼/링크 hover, focus 스타일 확인
5. `/scenarios/*` 페이지에서 스티키 CTA 숨김 동작 확인
6. 상황추천 필터 변경 후 "추천 다시 계산" 클릭 시 결과 유지/에러 fallback 확인
7. `/trends/[role]`, `/compare`에서 `LIVE/FALLBACK` 모드 전환 표시 확인
8. `/watchlist`에서 추가/삭제/전체삭제 및 새로고침 후 유지 확인
9. `/trends/[role]?topN=5|8|12`에서 Top N 반영 확인
