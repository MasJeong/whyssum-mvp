# 주의사항 및 확인 포인트

## 기능 동작

- 모바일 하단 탭/스티키 CTA는 `max-width: 720px` 구간에서만 노출됩니다.
- 비교 선택 UI는 최소 1개, 최대 4개로 제한되어 있습니다.
- 트렌드/상황추천의 role active 상태는 경로 prefix 매칭 기준입니다.
- 상황추천은 조건 변경 시 자동 재계산 + 수동 재계산 버튼이 함께 동작합니다.
- 추천 결과에는 카드별 "추천 근거"와 신뢰도 배지(High/Medium/Low)가 함께 표시됩니다.
- 추천 카드에는 Why now 문장과 트레이드오프(속도/안정성/확장성) 프로그레스바가 표시됩니다.
- 상황추천 저장 조건은 브라우저 localStorage 기반이며 직무별 최대 8개까지 유지됩니다.
- 같은 조건을 다시 저장하면 중복 생성 대신 저장 시각만 갱신됩니다.
- 상황추천 카드는 기본 요약 + 토글형 상세 뷰(자세히 보기/간단히 보기)로 동작합니다.
- 상세 뷰는 사용자가 빠르게 읽을 수 있도록 핵심 요약/상황 해설/실행 체크리스트 순서로 구성됩니다.
- 트렌드 페이지 Top N는 5/8/12 선택 가능하나, 실제 표시는 해당 role 데이터 개수 상한을 따릅니다.
- 관심리스트는 브라우저 localStorage 기반이며 기기/브라우저 간 동기화되지 않습니다.
- 브리핑은 현재 수동 큐레이션 데이터 기반이며 자동 수집 스케줄링은 미적용 상태입니다.
- 브리핑 목록/요약은 현재 필터 결과 기준이며 기본 정렬은 "영향도 우선 → 동일 영향도 내 최신순"입니다.
- 전역 포커스 가시성(`:focus-visible`)을 적용했으므로 키보드 탐색 시 윤곽선이 보여야 정상입니다.
- 비교/트렌드 테이블은 첫 번째 열이 고정(sticky)되며 가로 스크롤 시에도 기술명이 유지됩니다.
- 상황추천/비교 상단의 "한 줄 결론"은 빠른 판단 보조용이며 최종 결정은 상세 근거 확인이 필요합니다.
- 모바일 고정 CTA는 현재 화면 문맥(role)에 맞춰 이동 경로가 달라집니다.
- "자주 찾는 상황" 칩은 링크 동작으로 필터 초기값(teamSize/timeline/priority)을 URL로 전달합니다.
- 내비/CTA 아이콘은 의미 기반 SVG로 교체되었으므로 라벨과 아이콘 의미가 불일치하지 않는지 확인해야 합니다.

## 디자인/접근성

- 컬러 대비를 다크 배경/저밝기 기기에서 추가 검토 권장
- 모바일에서 하단 탭 + 스티키 CTA 동시 노출로 콘텐츠 가림이 없는지 확인 필요
- 체크박스 카드의 키보드 탐색 포커스 스타일(명확성) 추가 개선 여지 있음
- FAQ/랜딩 섹션 텍스트 길이가 늘어나 모바일에서 스크롤 피로가 없는지 확인 필요

## 보안/운영

- 기존 보안 미들웨어와 API rate limit은 유지됨
- 현재 rate limit은 메모리 기반이라 다중 인스턴스/재시작 환경에서 약함
- 운영 전 Redis 기반 분산 rate limit 전환 권장
- 성장 KPI 대시보드(`/growth`)와 `GET /api/growth-kpis`는 현재 메모리 이벤트 저장소 기준이라 서버 재시작 시 누적 수치가 초기화됩니다.
- `GET /api/trends/schedule`은 최근 실행 이력을 반환하며, 파일 저장 실패 시 메모리 모드로 자동 폴백됩니다.
- `TRENDS_ALERT_WEBHOOK_URL`이 설정되면 실패 또는 fallback 실행 시 운영 웹훅 알림을 시도합니다.
- GitHub 공개 API는 무인증 호출 시 rate limit이 낮아 트래픽 증가 시 fallback 빈도가 올라갈 수 있음
- 일부 repo가 일시 실패해도 전체 fallback으로 내려가지 않도록 부분 fallback이 적용되어 있습니다.
- npm/PyPI 보조 소스 호출 실패 시 해당 소스만 제외하고 계산합니다.
- 검색엔진 인덱싱 제어가 적용되어 `/api/*`, `/watchlist`는 기본 크롤링 대상에서 제외됩니다.
- 함수 주석 정책상 named function/method에는 JSDoc이 필수이므로 신규 코드 추가 시 누락 여부를 반드시 점검해야 합니다.

## QA 권장 시나리오

1. 모바일 뷰(360px~430px)에서 하단 네비/CTA 동작 확인
2. 비교 페이지에서 4개 초과 선택 차단 확인
3. `/trends/[role]`, `/scenarios/[role]` role 전환 active 상태 확인
4. 전체 페이지에서 버튼/링크 hover, focus 스타일 확인
5. `/scenarios/*` 페이지에서 스티키 CTA 숨김 동작 확인
6. 상황추천 필터 변경 후 "추천 다시 계산" 클릭 시 결과 유지/에러 fallback 확인
7. 상황추천 카드에서 신뢰도 점수/등급과 Why now 문장이 조건 변경에 맞게 갱신되는지 확인
8. 상황추천 카드의 트레이드오프 바(속도/안정성/확장성) 수치가 0~100 범위를 유지하는지 확인
9. 상황추천에서 조건 저장 후 새로고침/재방문 시 저장 목록이 유지되는지 확인
10. 다른 직무(`/scenarios/designer`, `/scenarios/pm`)로 전환 시 저장 목록이 직무별로 분리되는지 확인
11. 저장 조건이 8개를 넘을 때 오래된 조건이 정리되는지 확인
12. 마지막 선택 복원(직무별)이 기대대로 동작하는지 확인
13. 상황추천 카드에서 "자세히 보기" 클릭 시 정합성 해설/체크리스트/반영 규칙이 노출되는지 확인
14. "간단히 보기"로 되돌렸을 때 카드 높이와 레이아웃이 정상 복원되는지 확인
15. 상세 뷰에서 점수 구조(기본→조건 반영), 신뢰도, 리스크가 한 눈에 읽히는지 확인
16. 상세 뷰에서 추천 근거/장점/리스크가 확장 시에만 노출되어 기본 카드가 과밀하지 않은지 확인
17. `/trends/[role]`, `/compare`에서 `LIVE/FALLBACK` 모드 전환 표시 확인
18. `/watchlist`에서 추가/삭제/전체삭제 및 새로고침 후 유지 확인
19. `/trends/[role]?topN=5|8|12`에서 Top N 반영 확인
20. `/briefings`에서 role/impact/period 필터 조합별 결과 개수 확인
21. `/briefings` 상단 핵심 요약(중요 이슈/최근 7일/추천 직무)이 필터 결과와 일치하는지 확인
22. `/briefings` 빈 상태에서 "추천 필터로 다시 보기" 클릭 시 결과가 복구되는지 확인
23. `/briefings` 카드 액션 우선순위가 상황추천 > 트렌드 > 비교 > 원문 순으로 보이는지 확인
24. `/robots.txt`에 `/api/`, `/watchlist` disallow 규칙이 노출되는지 확인
25. `/api/recommendations`와 `/watchlist` 응답에 `X-Robots-Tag` 헤더가 포함되는지 확인
26. 키보드(Tab) 탐색 시 버튼/링크/입력 요소에 포커스 아웃라인이 보이는지 확인
27. 모바일에서 주요 액션 버튼/role pill의 터치 영역이 충분한지(오탭 없는지) 확인
28. `/compare`, `/trends/[role]`에서 가로 스크롤 중 첫 열 sticky 유지 및 hover 가독성 확인
29. `/scenarios/[role]`, `/compare`에서 한 줄 결론 배너가 현재 조건/선택 항목에 맞게 갱신되는지 확인
30. `/scenarios/[role]`의 "자주 찾는 상황" 칩 클릭 시 필터가 해당 값으로 즉시 반영되는지 확인
31. `/trends/[role]` 상단 상태 칩 문구가 사용자 관점(실시간 수집/안정 모드)으로 노출되는지 확인
32. 모바일에서 하단 탭 아이콘+라벨의 가독성과 활성 탭 구분이 명확한지 확인
33. 주요 CTA(홈/직무/브리핑/트렌드/상황추천)에서 아이콘과 액션 의미가 직관적으로 일치하는지 확인
34. Primary 버튼의 블루 계층과 신뢰도 배지(High/Medium/Low) 색상 대비가 충분한지 확인
35. 홈 구독 폼에서 유효하지 않은 이메일/중복 구독/성공 응답 문구가 기대대로 노출되는지 확인
36. `/api/subscribe` 응답에 `X-RateLimit-*` 헤더가 포함되는지 확인
37. 상황추천/비교의 "링크 복사", "요약 복사" 버튼이 실제 클립보드에 반영되는지 확인
38. 관심리스트에서 JSON 내보내기 후 가져오기 시 항목이 중복 없이 병합되는지 확인
39. 상황추천/비교/브리핑 로딩 중 스켈레톤이 표시되고 완료 후 콘텐츠로 전환되는지 확인
40. `/sitemap.xml`, `/feed.xml`, `/manifest.webmanifest`가 정상 응답되는지 확인
41. 동적 페이지(`/trends/[role]`, `/scenarios/[role]`)의 title/description/canonical이 직무별로 달라지는지 확인
42. 홈 페이지에 FAQ JSON-LD 스크립트가 포함되는지 확인
43. 광고 슬롯 노출 시 레이아웃 점프(CLS) 없이 고정 영역이 유지되는지 확인
44. `/advertising`, `/privacy` 페이지 링크가 푸터에서 정상 접근되는지 확인
45. `/briefings`에서 재방문 시 신규 브리핑 배지가 기대대로 표시되는지 확인
46. `/api/growth-events`에 잘못된 이벤트 바디 전송 시 400으로 안전하게 실패하는지 확인
47. `/api/growth-kpis?window=d1|d7|d30`가 정상 응답하고 잘못된 window는 400을 반환하는지 확인
48. `/scenarios/[role]?snapshot=...`, `/compare?snapshot=...` 진입 시 공유 상태가 복원되는지 확인
49. 관련 콘텐츠 섹션이 페이지 문맥에 맞는 링크 3개를 보여주는지 확인
50. 광고 슬롯 노출/클릭 이벤트가 중복 폭주 없이 기록되는지 확인
51. `/growth`에서 D1/D7/D30 카드와 상위 이벤트 목록이 모두 렌더링되는지 확인
52. 홈/직무/트렌드/상황추천/비교/인사이트 재진입 시 재방문 배지가 첫 방문에는 숨고 재방문 시에만 노출되는지 확인
53. `/api/trends/schedule` 응답에 최근 실행 `history`와 `persistenceMode`가 포함되는지 확인
