# 브리핑 재방문 허브 고도화 구현 계획서

## 목적

브리핑 페이지를 단순 기사 목록이 아니라, 사용자가 다시 방문해 의사결정을 이어가는 허브로 고도화한다.

## 배경 문제

- 현재 브리핑은 필터 + 카드 목록 중심으로, "지금 무엇을 먼저 봐야 하는지"가 약하다.
- 재방문 시 지난번 읽던 맥락(직무/영향도/기간)을 바로 복원하지 못한다.
- 브리핑 소비 이후 트렌드/비교/상황추천으로의 다음 행동 유도가 약하다.

## 목표

1. 첫 화면 3초 내에 "오늘 중요한 변화"를 파악할 수 있어야 한다.
2. 재방문 사용자는 마지막 필터 상태를 복원해 탐색 시작 시간을 줄여야 한다.
3. 브리핑 카드에서 다음 행동(트렌드/비교/상황추천) 전환율을 높여야 한다.

## 범위

### 포함

- 브리핑 상단 "오늘의 핵심 요약" 블록 추가
- 필터 상태 localStorage 복원/저장
- 카드 액션 우선순위 재배치(핵심 액션 우선)
- 빈 결과/로딩 문구를 행동 유도형으로 개선
- 경량 이벤트 로깅(클라이언트)으로 사용 흐름 측정

### 제외

- 외부 RSS/API 자동 수집 파이프라인
- 서버 DB 저장 기반 개인화
- 로그인/사용자 계정 기반 히스토리

## 정보 구조 개편

1. 헤더 영역
   - 현재 제목/설명 유지
   - 핵심 요약 카드(영향도 High 개수, 최근 7일 신규 항목 수)

2. 필터 영역
   - 직무/영향도/기간 유지
   - "최근 사용 필터 복원" 안내 문구 추가

3. 결과 영역
   - 카드 정렬: 영향도 high -> medium -> low, 동일 영향도 내 최신순
   - 카드 상단에 "왜 지금 봐야 하는지" 한 줄 추가(요약 기반)
   - 주요 액션 우선순위: 상황추천 > 트렌드 > 비교 > 원문

4. 빈 상태
   - 단순 "없음" 대신 완화 제안 버튼 제공
   - 예: 기간 90일로 확장, 영향도 전체로 변경

## 데이터/로직 설계

### API (`src/app/api/briefings/route.ts`)

- 기존 필터 파라미터 유지
- 추가 응답 필드
  - `summary.highImpactCount`
  - `summary.recentCount7d`
  - `summary.recommendedRole` (가장 항목이 많은 role)

### 클라이언트 (`src/components/briefing-board.tsx`)

- localStorage 키: `whyssum:briefing:lastFilters`
- 저장 대상: `role`, `impact`, `periodDays`
- 마운트 시 복원, 변경 시 debounce 저장
- 카드 우선순위 정렬은 클라이언트에서 1차 적용(후속으로 API 정렬과 통합 가능)

## 측정 지표 정의

- `briefing_view`: 브리핑 화면 진입
- `briefing_filter_change`: 필터 변경
- `briefing_card_action_click`: 카드 액션 클릭(대상 action 포함)
- `briefing_empty_state_recover`: 빈 상태에서 추천 액션으로 복구

> 1차는 콘솔/경량 함수 호출 형태로 시작하고, 이후 서버 전송 이벤트로 확장한다.

## 구현 단계

1. API 응답 확장
2. 브리핑 보드 상태 복원(localStorage)
3. 상단 핵심 요약 UI 추가
4. 카드 우선순위/액션 순서 개편
5. 빈 상태 액션 강화
6. 경량 이벤트 훅 추가
7. 문서 최신화(`README.md`, `review_docs/changed-files.md`, `review_docs/cautions.md`)

## 수용 기준

- 재방문 시 마지막 필터 상태가 자동 복원된다.
- 상단 요약에서 high 영향도 개수와 최근 7일 항목 수를 확인할 수 있다.
- 카드 액션의 기본 강조가 상황추천으로 이동한다.
- 빈 상태에서 제안 액션 클릭 시 결과가 다시 노출된다.
- `npm run lint`, `npm run build` 통과
