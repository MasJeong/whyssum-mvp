# 복원/볼륨제어 스트레스 검증 시나리오

## 목적

- 시나리오 복원, 브리핑 복원, 추천 카드 볼륨 제어가 반복 사용에서도 안정적으로 동작하는지 확인한다.
- 5회 반복 점검에서 상태 유실/오복원 0건을 목표로 한다.

## 범위

- 시나리오 복원: `src/components/scenario-explorer.tsx`
- 브리핑 필터 복원: `src/components/briefing-board.tsx`
- 추천 카드 볼륨 제어(상위 3개/전체): `src/components/scenario-explorer.tsx`

## 사전 조건

- 개발 서버 실행: `npm run dev`
- 브라우저 캐시 및 localStorage 접근 가능한 상태

## 테스트 시나리오 (1회차)

1. 시나리오 복원
   - `/scenarios/backend` 진입 후 팀 규모/일정/우선순위를 변경한다.
   - 페이지 새로고침 후 변경한 값이 유지되는지 확인한다.
   - `/scenarios/designer`로 이동 후 다시 `/scenarios/backend`로 돌아와 역할별 마지막 선택 복원이 맞는지 확인한다.

2. 저장 스냅샷 적용
   - 현재 조건을 저장한 뒤 다른 조건으로 바꾼다.
   - 저장한 스냅샷 칩을 눌러 원래 조건이 정확히 복원되는지 확인한다.
   - 공유 링크(`?snapshot=`)로 재진입해 조건이 적용되는지 확인한다.

3. 추천 카드 볼륨 제어
   - 추천 결과가 4개 이상인 조건에서 기본 노출이 상위 3개인지 확인한다.
   - `전체 보기` 클릭 시 전체 카드가 보이는지 확인한다.
   - `상위 3개만 보기`로 되돌아올 때 카드 수가 정확히 3개인지 확인한다.

4. 브리핑 필터 복원
   - `/briefings`에서 직무/영향도/기간 필터를 변경한다.
   - 새로고침 후 마지막 필터가 복원되는지 확인한다.
   - 다른 페이지 이동 후 재진입 시에도 복원 상태가 유지되는지 확인한다.

## 반복 규칙

- 위 1~4 시나리오를 연속 5회 반복한다.
- 각 회차마다 성공/실패를 아래 양식으로 기록한다.

## 종료 기준

- 5회 반복에서 치명 이탈(상태 유실/오복원/토글 오작동) 0건이면 통과.
- 실패 1건 이상이면 재현 경로를 비고에 남기고 우선 수정 티켓으로 전환한다.
