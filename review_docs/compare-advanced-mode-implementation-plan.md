# 비교 고급 모드 구현 계획서

## 목적

비교 기능의 기본 사용성(간결함)을 유지하면서, 파워 유저가 더 많은 후보를 한 번에 검토할 수 있도록
`기본 4개`와 `고급 8개` 모드를 제공한다.

## 배경 문제

- 현재 비교 선택 상한이 4개로 고정되어 있어, 탐색 폭이 넓은 사용자는 여러 번 왕복해야 한다.
- 반대로 기본 화면에서 과도한 항목을 허용하면 모바일 가독성과 판단 속도가 떨어질 수 있다.

## 목표

1. 기본 모드는 `최대 4개`를 유지해 빠른 판단 흐름을 지킨다.
2. 고급 모드는 `최대 8개`까지 확장해 탐색 폭을 제공한다.
3. 상한 도달 시 사용자에게 명확한 안내를 제공한다.

## 범위

### 포함

- 비교 모드 토글 UI(기본/고급)
- 선택 상한 동적 변경(4/8)
- 선택 상한 도달 시 안내 문구
- 고급에서 기본으로 전환 시 자동 정리(4개로 컷)

### 제외

- 비교 항목을 스크롤 가상화로 최적화
- 역할별 모드 선호도 영구 저장
- 비교 결과 그래프 시각화 추가

## 설계

1. 상태
   - `isAdvancedMode: boolean`
   - `maxSelectable = isAdvancedMode ? 8 : 4`

2. 상호작용
   - 기본 모드: 4개 초과 선택 차단 + 안내
   - 고급 모드: 8개 초과 선택 차단 + 안내
   - 고급 -> 기본 전환 시 현재 선택이 4개 초과하면 앞 4개 유지

3. 접근성
   - 모드 토글에 `role=tablist/tab`, `aria-selected` 적용
   - 비활성 체크박스는 `disabled`로 명확히 차단

## 수용 기준

- 기본 모드에서 5번째 선택은 차단되고 안내 문구가 보인다.
- 고급 모드에서 최대 8개까지 선택 가능하다.
- 고급 모드에서 9번째 선택은 차단된다.
- 고급 모드에서 8개 선택 후 기본 모드로 전환하면 4개로 정리된다.

## 리스크 및 대응

- 8개 비교 시 테이블 가로 폭 증가
  - 대응: 기존 가로 스크롤/첫 열 고정 동작을 유지해 읽기 가능성 확보
- 선택 자동 정리 시 사용자 혼란
  - 대응: 전환 직후 안내 문구로 변경 사실을 즉시 고지
