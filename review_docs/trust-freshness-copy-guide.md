# 신뢰도/신선도 문구 정렬 가이드

## 목적

- `live/fallback`, `confidenceScore` 문구를 화면마다 일관되게 노출해 오해를 줄인다.
- "실시간 정확도"로 과장 해석되지 않도록 의미 범위를 명확히 고정한다.

## 핵심 원칙

1. `live/fallback`는 수집 상태다.
   - 의미: 데이터 출처/수집 성공 여부
   - 비의미: 추천 정확도 보장

2. `confidenceScore`는 휴리스틱 합성값이다.
   - 의미: 적합도 + 조건 근거 + 트렌드 신호를 조합한 판단 보조 지표
   - 비의미: 절대 정답 확률

3. 사용자 액션 유도 문구는 항상 함께 둔다.
   - 예: "점수가 애매하면 조건을 바꿔 재계산하세요"

## 화면별 권장 문구

### 트렌드 (`/trends/[role]`)

- 상태 라벨
  - `live`: "live (외부 지표 기반 최신 수집)"
  - `fallback`: "fallback (외부 수집 실패 시 기본 데이터)"
- 신뢰도 보조
  - "신뢰도 점수는 데이터 상태/일관성 기반 휴리스틱입니다."

### 상황추천 (`/scenarios/[role]`)

- 안내 버튼
  - "신뢰도 계산 안내"
- 확장 문구
  - "신뢰도 점수는 적합도, 조건 매칭 근거, 트렌드 신호를 합성한 휴리스틱 지표입니다."
  - "live/fallback은 데이터 수집 상태를 의미하며 추천의 절대 정확도를 보장하지 않습니다."

### 비교 (`/compare`)

- 표 헤더/도움말
  - "신뢰도는 상대 비교 보조 지표이며 비용/리스크와 함께 판단하세요."

## 금지 문구

- "실시간으로 정확한 추천"
- "신뢰도 점수 = 성공 확률"
- "fallback이므로 품질이 낮다"(단정)

## QA 체크포인트

- 동일 개념(`live/fallback`, `confidenceScore`)이 화면마다 같은 의미로 읽히는가
- 신뢰도 설명 문구가 액션 유도(재계산/비교/근거 확인)와 연결되는가
- 오해 가능 표현(실시간, 정확도 보장)이 남아있지 않은가
